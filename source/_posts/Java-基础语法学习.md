## 语法基础
1. Java语法格式：
```
package 包路径;
public class 类名 {
    public statid void main(String[] args) {
        程序代码
    }
}
```
2. main方法是执行程序的入口，当开始运行当前程序时，Java虚拟机会按顺序由上至下执行main方法内部的程序代码。
定义标识符不能与关键字冲突，业界定义标识符时遵守一定规范大致如下：
* 包名一律使用小写字母
* 类名中每个单词首字母都要大写
* 变量名与方法名使用“驼峰命名法”
* 常用名所有字母大写，如有多个单词则使用“_”连接，如USER_ID
注释分为三种：

单行注释
```
 //我是一个单行注释
```
多行注释
```
/*我是一个多行注释*/
```
文档注释
```
/**
 *这是啥意思
 *这又是啥意思
 *我是文档注释
 */
```
变量的声明：需要注意的是局部变量声明后必须进行初始化。不然编译器会报错。
Java的基本数据类型有八种：byte、short、int、long、float、double、char、boolean。
**注：**需要注意的是不同类型变量赋值时的数值溢出问题，如下：
```
public class Example {

    public static void mian(String[] args) {
    
        long index = 300; //300(int类型)隐式转换成long
        int anther = index; //把index变量赋值给anther，编译器报错
        int anther = (int)index; //强制转换为int不会报错，可能会出现值的问题
    }

}
```
## 流程控制语句
流程控制语句主要分为3种：顺序结构、分支结构和循环结构。
1. 顺序结构就是我们常编写的代码，从上到下执行的结构，这个比较简单就不详细介绍了。
2. 分支结构分为两大类：if语句和switch语句。
   1)  if语句：
```
if(判断条件){
    代码块（判断条件成立后执行）
}
如下：
int age = 16;
if(age >= 16){
    System.out.println("小明成年了");
}
```

2) if else 语句：
```
if(判断条件){
    A 代码块（判断条件为true执行）
}else{
    B 代码块（判断条件为false执行）
}
如下：
int age = 16;
if(age >= 16){
    System.out.println("小明成年了");
}else{
    System.out.println("小明未成年了");
}

```
**注：Java中有一种特殊的运算叫三元运算，执行的流程和if...else语句类似语法如下：**
```
变量类型 v = 判断条件 ？ 表达式1 ： 表达式2
注：表达式1和表达式2的结果的数据类型需要保持一致。
```
判断的流程是：
> 执行判断条件，判断条件是true还是false。
> 如果条件为true则运算表达式1，然后把表达式1的结果值作为三元运算的结果赋值给变量v。
> 如果条件为false则运算表达式2，然后把表达式2的结果值作为三元运算的结果赋值给变量v。

3) if...else if...else 语句
```
if(判断条件A){
    A 代码块 （判断条件A值为true，执行）
}else if(判断条件B){
    B 代码块 （判断条件B值为true，执行）
}else if(判断条件C){
    C 代码块 （判断条件C值为true，执行）
}else{
    D 代码块 （判断条件A、B和C值为false，执行）
}
```
4) switch语句：
Java中提供了switch语句来来简化过于啰嗦的分支语法，语法如下：
```
switch (分支表达式) {
    case 目标值 A:
        执行语句A;
        break;
    case 目标值 B:
        执行语句B;
        break;
    case 目标值 C:
        执行语句C;
        break;
    case 目标值 D:
        执行语句D;
        break;
    ......
    ......
    default:
        默认执行语句;
}
```

3. 循环结构语句有三类：
    * while循环
```
while (循环条件){
    循环体执行语句;
    循环体执行语句;
    ...
}
```
这里需要注意的是，在循环体中要给出正确的跳出循环的条件，否则这个程序就进入了死循环！！！
* do...while循环
```
do {
    循环体执行语句
}while(循环条件);

//这种循环首先会执行一次循环体，然后判断条件
```
* for 循环
```
for (初始表达式; 循环条件; 操作表达式){
    循环体执行语句
}

for(int i = 0; i < 10; i++){
    System.out.println("Hello World");
}
```

以上就是流程控制语句的基本介绍；

## 数组和方法
1. 数组时Java中极为常用的数据类型，声明数组变量时需要明确定义数组中存储的数据类型，如果数组中存储int类型数据就要声明为：int[]。Java要求声明数组时必须指定元素的数据类型，并且数组中的元素必须与该类型一致，否则会导致语法错误。如下初始化：
```
int[] ary;
ary = new int[5];
double[] ary2 = new double[5];
```
还可以如下初始化：
```
int[] ary1 = {100, 200, 300, 400};
int[] ary2 = new int[] {100, 200, 300, 400};
```
**注：**
>  整型数组元素的默认值为0
    浮点型数组元素的默认值为0.0
     boolean类型数组元素的默认值为false
     char类型元素的默认值看不到，也是有的是Unicode编码为0的字符
     String类型数组默认值为null
Java数据类型分为两大类：基本数据类型和引用数据类型。

2. 方法是程序的核心，Java中定义一个方法的具体格式如下：
```
修饰词 返回值类型 方法名 （参数类型 参数名， 参数类型 参数名）{
    方法体执行语句
}
如以下：
public static void arraySort (int[] ary){
    ... ...
    ... ...
}

```
1. 修饰词：有访问权限修饰词public，静态修饰词static，最终修饰词final等。
2. 返回值类型：用于限定方法返回值的数据类型。
3. 参数类型/参数名：用于接受方法调用者传递的参数数据。
4. 方法体：调用方法将会执行的代码段。
5. return：return关键字用于接受方法的执行并返回方法执行的最终结果。
6. 返回值：方法执行的最终结果。

## 类和对象
 对象是用程序虚拟出来的物体，包含物体的属性和行为，它本质上是内存中存储的一份数据。类是对象的“设计图纸”，是设计对象的具体手段，它本质上是对象的数据类型。所以想要使用对象，就必须先定义类。
> 类中直接声明的变量叫做成员变量，方法内部声明的变量叫做局部变量。对于未初始化的成员变量，在创建对象时JVM会自动为它们初始化，并根据类型赋予它们不同的初始值。

JVM 在运行时会向操作系统申请一块内存空间，它在管理这块内存空间时会将其划分成若干个不同的数据区域。
### 栈：
以方法为单元来存储数据。在调用任何方法时，JVM都会为此方法创建一个名为栈帧的空间，用于存储该方法内部的数据，再将此栈压入栈内，该过程叫做入栈。方法调用结束时，JVM会将此栈帧从栈内弹出，此过程叫做出栈。
> 调用方法1时，JVM会为方法1创建一个栈帧，用于存储方法1内部的数据，并将此栈帧压入栈内，同样方法2也是。此时由于栈帧1已经位于栈底，所以栈2停留在栈1的上方。可以用下图表示：
![入栈过程](https://ws3.sinaimg.cn/large/006tNc79ly1fqjhhpg3klj30vm0r23zk.jpg)

###  堆：
专门用来存放数组和对象数据。由于数组和数组之间、对象和对象之间都是彼此独立的数据，它们之间没有顺序，被零散地存放在堆这个内存区域内，无论是数组还是对象，它们占据的那部分空间都有一个明确的地址，便于外界访问它们。如下图：
![堆结构](https://ws3.sinaimg.cn/large/006tKfTcly1fqji2oa8urj30va0pumyq.jpg)

下面我们就以赛车手为例，详细阐述对象的创建过程，代码如下：
```
public static void main (String[] args) {
    Car car1 = new Car();
    car1.brand = "奔驰";
    car1.color = "红色";
    car1.maxSpeed = 350;
    car1.number = "京A00001"

}
```
在main 方法之内的第一行，我们创建了对象，代码如下：
```
Car car1 = new Car();
```
运行代码时会分为3个步骤：
* 首先执行的是等号左侧的部分，即声明变量的代码 Car car1 。执行程序到此时，由于运行的是main方法，所以JVM会为main方法创建一个栈帧，然后入栈。又因为声明了变量car1 ，所以JVM会在main方法的栈帧里开辟一块空间，用于存放car1变量的值。这个过程如下图所示：
![](https://ws4.sinaimg.cn/large/006tKfTcly1fqjyjvx2wlj30u60m03z3.jpg)

* 接下来是执行等号右边的部分，即创建对象的代码块“new Car()” ，这时候JVM在堆中为新对象开辟一块空间，假设该空间的地址为0x1001。然后JVM在此空间内开辟了4个字空间，用于存储对象的4个属性值，由于此时还未对属性赋值，所以JVM先将它们设置为初始值，分别存于4个字空间中，整个过程如下图：
![](https://ws2.sinaimg.cn/large/006tKfTcly1fqjzdk4vd9j31780l0dhp.jpg)

* 然后是执行赋值操作，即“=”。此时JVM将对象的地址存入car1变量所引用的空间内。这个过程如下图所示：（注红色）
![](https://ws2.sinaimg.cn/large/006tKfTcly1fqjzc1v5lkj316y0myq4x.jpg)

至此，栈和堆中的数据产生了联系，栈中没有直接保存对象数据，仅仅是储存了对象的地址，用地址来引用对象，所以说对象是引用类型的数据。
执行完这行创建对象的代码后，又为对象的属性进行了赋值，代码如下：
```
car1.brand = "奔驰";
car1.color = "红色";
car1.maxSpeed = 350;
car1.number = "京A00001"
```
此时JVM就将这些值存入各个属性在堆中所占的空间内，如下图所示：
![](https://ws4.sinaimg.cn/large/006tKfTcly1fqjzl1w3m3j318c0kwac7.jpg)

以上就是对象创建过程中，JVM内存中的数据变化过程。每个方法在执行时都对应一个独立的栈帧，栈帧内部存储的是该方法的参数和局部变量。**注：**通过类创建对象的过程也叫实例化，对象也叫类的实例。

###  构造方法
* 构造方法又称构造器，是一种特殊的方法，是专门用来初始化对象，在对象创建是为对象的属性初始化数据的。构造方法的规则如下：
    > 1. 方法名必须与类名相同。
    > 2. 方法不能有返回值。
    > 3. 方法名前不能声明返回值类型，void也不行。

下面用代码说明：
```
package demo;
class Car {
    //品牌
    String brand;
    //颜色
    String color;
    //最高时速
    int maxSpeed;
    //车牌号
    String number;
    
    //构造方法
    public Car(String b, String c, int s, String n){
        brand = b;
        color = c;
        maxSpeed = s;
        number = n;
        }
    void run(){
        System.out.println("车牌号：" + number);
        System.out.println(brand + color);
        System.out.println("最高时速：" + maxSpeed);
        }
}
```
接着我们可以用上面的构造方法来创建Car对象。如下代码：
```
package demo;
public class RacingGame{
    public static void main(String[] args) {

    Car car = new Car("奔驰","红色",350,"京A00001");
    car.run();
    }
}

```
****
**注：**每个类都有构造方法，只有通过构造方法才能实例化对象，但是构造方法却不是必须要声明的，对于未声明的构造方法，在编译时JDK会给它增加一个无参数的构造方法，通常称这个无参数的构造方法为默认的构造方法，如果我们显式地声明了构造方法，那么在编译的时候JDK就不会自动增加默认构造方法了。

### 重载
（名称相同参数不同**Object-C不支持**）
1. 构造方法的重载：为一个类声明多个名称相同，参数不同的构造方法，叫做构造方法的重载。这样设计的目的是方便调用者，调用者可以根据自己掌握的信息，选择合适的构造方法来创建对象。
2. this关键字用来指代当前对象，有三种用法：
    > 调用当前对象的成员变量
    > 调用当前对象的成员方法
    > 调用当前类的构造方法 **注：**只能在构造方法的第一行，通过this关键字调用其他的构造方法
    
    * 同理也有方法的重载，这个比较简单就不详细说明了。
    

## 面向对象的三大特征
面向对象的三大特征：封装、继承和多态。这三大特征是践行面向对象编程思想的指导性意见，是设计程序时要遵循的基本原则。
### 封装：
所谓的封装，就是将该显示的显示，该隐藏的隐藏，Java中控制显示与隐藏需要使用访问修饰符，常用的private和public，一般声明成员变量时使用private，然后对private的成员变量增加set和get封装方法。在set方法中进行封装去除不合理的值。
### 继承：
继承这个好理解，基本上所有的面向对象的程序设计都有继承的概念。下面说下Java继承的特点。
* 某个类只能有一个直接的父类，这种规则叫做单继承。Java中不允许出现多继承。
* 允许出现多层继承的情况。
* 多个类可以继承同一个父类。
* 子类不能继承父类私有的内容，不能调用服了私有的成员变量、成员方法和构造方法。

通过继承，子类可以重写父类的方法。子类可以重写父类的成员方法，但是不能重写父类的构造方法。同this关键字一样。super关键字可以调用父类的成员变量、成员方法和构造方法。
**注：，创建子类对象时，JVM会先调用父类的构造方法（并没有创建父类对象，因为new的是子类不是父类）。这是隐式调用的。通过super关键字显示调用父类的构造方法，这行代码必须写在子类构造方法的第一行**

### 多态：
表现是一个父类表现出了多种不同的状态，这就是多态。其实就是父类的引用指向子类对象。
    在Java中，在运用多态的思想设计程序时，由于使用父类指代子类对象，这时父类和子对象类型是可以相互转换的。这时向上类型转换，Java中是默认支持的，但是一旦将对象的子类型转换成父类型，就只能对他调用父类型的属性和方法，此时如果继续调用子类型的属性和方法，就会提示编译器错误。
    
```
package demo;
class Persom {
    void drive(Vehicle v) {
    v.run();
    v.unload();//这里是子类的方法会报错
    }
}

```
这时需要将Vehicle类对象强制转换成子类Truck类型，前提是传入的对象是Truck对象。
```
package demo;
class Persom {
    void drive(Vehicle v) {
        Truck t = (Truck)v;
        t.run();
        t.unload();//这里是子类的方法会报错
        }
}
```
## 抽象类于接口
### 抽象类：
Java中，可以通过abstract关键字将成员方法声明为抽象方法，这样的方法不用具体实现，无需声明方法体，包含抽象方法的类叫做抽象类，在定义抽象类的时候也需要使用关键字abstract对类进行修饰。

```
package demo;
abstract class Vehicle {
    //行驶
    void run(){
        start();
        speedUp();
        stop();
    }
    //启动
    abstract void start();
    //加速
    void speedUp(){
        System.out.println("加大油门");
    }
    //停止
    void stop(){
        System.out.println("刹车停止");
    }
}
```
抽象方法的具体实现是在子类中完成的，子类必须实现父类声明的抽象方法，除非子类也是抽象类不需实现。
定义抽象类的规则：
> 抽象类和抽象方法必须使用关键字abstract来修饰，抽象方法不能有方法体。
> 抽象类中可以没有抽象方法，但是一个包含抽象方法的类必须被声明为抽象类。
> 子类若是抽象类，则可以不实现父类的抽象方法，子类不是抽象类，必须要实现父类的抽象方法。
> 抽象类不能被直接实例化，如下
```
abstract class A {

}

class B {
    public static void main(String[] args) {
        A a = new A();//会报错的
    }
}
```
> 抽象类有构造方法，但是不能直接调用其构造方法来实例化抽象类，必须在子类中调用，如下：
```
abstract class A {
    public A () {

    }
}

class B extends A{
    public B(){
        super();
    }
}
```
### 接口
接口可以看作是特殊的抽象类，它内部声明的方法都是抽象方法，所以接口抽象的更为彻底。接口需要采用interface
关键字进行定义，并且需要有专门的类来实现接口中的抽象方法，这样的类叫做实现类，它们采用implements关键字来实现接口，如以下代码：
```
//定义名称为A的接口
interface A {
    abstract void amethod();
}
//定义名称为B的接口，让它实现接口A
class B implements A {
    void amethod(){
        ...//代码
    }
}

```
一个类只能继承一个父类，但是却可以实现多个接口，这时接口最大的优点。
定义接口的规则如下：
1. 接口中可以定义属性，但是定义属性时必须初始化该属性，并且接口中的这个属性值不允许修改，因此也叫常量。
```
interface A {
    int i = 9;
}
```
2. 接口定义的方法全部是抽象方法，所以在方法前的abstract关键字可以省略，如下：
```
interface A {
    void aMethod();
}
```
3. 接口可以继承接口，但是不能继承类，并且一个接口可以继承多个接口，如下：
```
interface A {
}

interface B {
}

interface C extends A, B{
}
```
4. 接口的实现类若不是抽象类，就必须实现接口中的所有方法，参考如下：
```
interface A {
    void mehtod1();
    void mehtod2();
    void mehtod3();
}

class B implements A {
    public void method1() {

    }
    public void method2() {

    }
    public void method3() {

    }
}
```
5. 接口的实现类若是抽象类，则可以选择性地实现接口中的任意多个方法，参考如下：
```
interface A {
    void h1();
```

## 对象的管理
包：
在进行一个正式的商业项目开发时，业务往往比较复杂，需要设计大量的类，为了便于查找和管理这些类，通常会按照功能的不同将这些类存放在不同的包中。包就是用来管理类的“文件夹”。
访问修饰符：
在Java中，我们可以对类、成员变量和成员方法的访问进行十分精细的管理。对于这三者的访问一共有4个级别分别是：private、default、protected和public。默认是defaul。
**注：public可以修饰类，private和protected不能修饰类。**

对4个访问级别的详细介绍如下：
1. private级别：
> 被修饰符private修饰的成员变量，成员方法为private级别，该级别的成员变量，成员方法只能被当前类的其他成员所访问。
2. default级别：
> 未经访问修饰符修饰的类、成员变量和成员方法为default级别，这个级别的内容可以被当前包下的其他类所访问，无法被当前包之外的类所访问。
3. protected级别：
> 被protected修饰的成员变量，成员方法为protected级别，该级别的成员变量和成员方法可以被当前包下其他的类所访问，也可以被任意包下改类的子类访问。
4. public级别：
> 被修饰符public修饰的类、成员变量和成员方法为public级别。该级别的类、成员变量和成员方法可以被任何包下的任何类所访问。

垃圾回收：
JVM会定时扫面堆内存，找出并删除该删除的数据，从而回收内存空间，这个过程就是垃圾回收。
Java采用的是自动的垃圾回收机制。在程序中我们也可以通过调用“System.gc()”来通知JVM立即进行垃圾回收，但是具体回收的时间，还是由JVM来决定，所以我们无法准确地控制垃圾回收时机。
一般不建议程序员调用“System.gc()”，因为JVM是按照一定的规律，对内存若干区域进行完整的扫描。手动调用会打破JVM的扫描规律，这会极大地消耗JVM的运行能力，大大地降低程序运行的效率。所以只需让JVM自动进行垃圾回收即可。
